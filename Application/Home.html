1. 個人簡介 / 職業背景

嗨，我是 pinyi


大概七年前開始接觸後端工程，從一開始單純覺得「寫程式很好玩」，到後來發現自己喜歡研究系統背後的邏輯和細節。 很多人不會注意的地方，其實正是系統能不能穩定運作的關鍵，比如效能、資安、架構設計這些事。


現在我在宏願數位股份有限公司擔任後端工程師，主要用 Golang、Docker、PostgreSQL、MongoDB、Redis、NATS、Elasticsearch 等技術， 負責支付系統、內容管理系統、帳號系統，以及智慧 NPC 服務的開發與維護。


除了寫程式，我也會從需求討論、系統設計、服務整合到效能優化一路跟進，確保系統既穩定又好維護。

2. 申請動機

我一直覺得，AI 在後端領域的應用才剛開始，真正的價值遠比我們現在看到的更多。


現在的系統早就不是 CRUD 這麼簡單，需要更智慧化的能力，才能真正幫使用者解決問題。 我想透過這門課，從數學基礎和神經網路原理開始，一步步學到資料處理、模型訓練，最終能自己完成一個 AI 應用服務。


對我來說，這不只是多一個技能，而是一次角色升級，從「後端工程師」成長為能把 AI 和後端緊密結合的人。無論是跟 AI 團隊合作，還是自己做專案，都能打造更智慧、更好用的系統，讓產品有更強的競爭力，也替自己的職涯打開更多可能性。

3. 想解決的問題

我家有一隻呼吸道狀況不太好的貓。


為了追蹤牠的健康，我幾乎每天都要手動數呼吸次數，還要記錄牠打噴嚏、咳嗽的頻率。

這件事很花時間，也容易漏記或數錯，但這些數據對醫生判斷病情又很重要。


我想做一個能自動記錄的工具，用影片計算呼吸次數，或用聲音偵測噴嚏和咳嗽。 這樣不只節省時間，數據也更精準，還能在異常時發出提醒。 這不僅能解放雙手，還能在第一時間發現異常，幫助我更安心地照顧牠。


對我來說，這不只是個技術挑戰，也是一個能直接改善我和牠生活品質的解決方案。

4. 學習時間安排

平日我每天會安排大約 3 小時學習，週末則是 5–8 小時，加起來每週至少 25 小時。


我知道同時兼顧工作和學習會很累，但我打算用穩定的投入和進度，證明自己是真的下定決心去完成這件事。

5. 處理負面情緒的經歷

有一次，我接手了一個多年沒更新、且沒有文件的老系統。


一開始客戶的問題一個接一個，忙得不可開交，我甚至懷疑自己撐不下去。 後來我想，不如換個方式應對，與其被問題追著跑，不如我先建立自己的節奏。


我先把所有已知問題列出來，再一行一行讀程式碼做筆記，並定期跟同事做 Code Review。 幾個月後，我不只可以快速定位問題，維護效率也提升了好幾倍，突發狀況的次數也大幅下降。


這讓我更確信，只要方法對，壓力也可以變成推動自己成長的動力。

6. 為什麼要學基礎模型的實作

我不期望自己能做到像 OpenAI 那樣全方位的模型。


但如果能掌握基礎模型的實作，就能在專案中選用最合適的模型，並針對特殊需求進行最佳化，打造客製化的方案。 雖然整體能力比不上 OpenAI，但在特定任務或獨特需求上，仍有機會達到更好的效果，同時減少對 OpenAI API 的依賴與成本。
